# 开发路线图

## 1. 路线原则

- 先业务闭环，再扩展通道
- 先统一引擎，再补充提供商
- 先文本客服主链路，再逐步叠加语音与 RTC
- 每个阶段都包含测试、文档与验收

## 2. 阶段规划

### 阶段 1：文档与文本客服闭环

目标：

- 完成业务、架构、模块、API、路线图、进展控制文档
- 建立项目骨架
- 完成文本客服、Session、RAG、LLM 编排、人工协同基础能力

依赖：

- 无

交付物：

- 文档基线
- 可运行 API 服务
- 文本客服与知识库接口
- 基础测试

### 阶段 2：业务工具与运营能力

目标：

- 完成业务工具调用
- 完成 Prompt/策略/指标/会话管理接口
- 完善路由与手工接管机制

依赖：

- 阶段 1 完成

### 阶段 3：语音客服

目标：

- 完成 ASR/TTS 抽象与默认提供商
- 完成语音请求 API
- 补齐语音异常流程测试

依赖：

- 阶段 1/2 的统一引擎与会话管理

### 阶段 4：RTC 实时通话

目标：

- 完成房间生命周期管理
- 完成 WebSocket 事件协议
- 支持打断、超时、结束、人工转接

依赖：

- 阶段 3 语音能力

### 阶段 5：交付收尾

目标：

- 执行完整性检查
- 修正文档与实现偏差
- 补全部署与测试说明

## 3. 联调顺序

1. 文本 API 与 Session
2. RAG 与知识库管理
3. 工具调用与路由策略
4. 人工转接与运营接口
5. 语音 API
6. RTC WebSocket

## 4. 测试顺序

1. 单元测试：路由、RAG、人工转接
2. 集成测试：文本客服、知识库管理、业务工具
3. 链路测试：语音客服闭环
4. 链路测试：RTC 通话状态切换与打断

## 5. 验收标准

- 所有核心接口可运行
- 文档与实现一致
- 关键链路存在自动化测试
- 进展控制文档真实反映项目状态
